Designing for Clarity

Great design is invisible. It guides users effortlessly through content, creating a seamless experience that feels natural. Every spacing decision, every font pairing, and every color choice contributes to this invisible architecture.

The Role of Typography

Typography is the backbone of any design system. The right type scale creates a clear hierarchy that helps readers scan and absorb information quickly. We use mathematical ratios to ensure consistency across all screen sizes.

Key Principles

  • Establish a clear visual hierarchy with consistent heading sizes
  • Use adequate line height for comfortable reading (1.5–1.8 for body text)
  • Maintain optimal line length (50–75 characters per line)
  • Choose font pairings that complement each other

"Good typography is invisible. Bad typography is everywhere."

Color & Spacing

A well-defined color palette and spacing system are the foundation of visual consistency. AI-driven adjustments ensure that padding and margins adapt to your chosen fonts for optimal legibility.

Color Theory in Practice

Understanding how colors interact helps create harmonious palettes that reinforce your brand identity.

Systematic Spacing

Consistent spacing creates rhythm and helps users navigate content without cognitive overload.

Implementation Steps

  1. Define your base font size and scale ratio
  2. Generate a color palette from your brand color
  3. Set up spacing tokens based on your type scale
  4. Apply semantic naming for easy Oxygen Builder integration
  5. Test across devices and adjust as needed

Oxygen Builder Integration

There are 3 ways to add this generated CSS into Oxygen Builder so that all elements respect your design tokens. Choose the method that fits your workflow.

1

Global Stylesheet (Recommended)

Loads on every page. Best for site-wide design tokens.

  1. In WordPress admin, go to Oxygen > Stylesheets
  2. Click Add Stylesheet and name it Design System Tokens
  3. Paste the CSS below into the editor
  4. Click Save

The CSS is stored in wp-content/uploads/oxygen/css/universal.css and loaded globally.

2

Code Block in a Template

Scoped to a template and its child pages. Good for multi-brand sites.

  1. Open your catch-all template (or any template) in Oxygen's Visual Editor
  2. Add element: +Add > Basic > Code Block
  3. Select the CSS tab in the Code Block properties
  4. Paste the CSS and save the template

Every page inheriting this template will use the design tokens. The Code Block position on the page doesn't matter for CSS.

3

WordPress Customizer (Fallback)

Quick option if you don't want to open Oxygen at all.

  1. Go to Appearance > Customize > Additional CSS
  2. Paste the CSS and click Publish

Injected in a <style> tag before </head>. Site-wide but lower specificity than Oxygen stylesheets.

Don't forget Google Fonts

The CSS uses Google Fonts that must be loaded on your site. Add this to your Oxygen template's Code Block > Header, or use a plugin like OMGF to self-host them.


                        
                    


                    
                    

                        
Using Variables in Oxygen

                        
Once the CSS is loaded, reference the variables anywhere in Oxygen's visual controls:

                        

                            

                                
Font Family

                                var(--font-heading)
                            

                            

                                
Font Size (e.g. H2)

                                var(--step-4)
                            

                            

                                
Brand Color

                                var(--color-brand)
                            

                            

                                
Spacing

                                var(--spacing-base)
                            

                            

                                
Border Radius

                                var(--radius-base)
                            

                            

                                
Heading Margin

                                var(--heading-margin-top)
                            

                        

                        
In Oxygen's element settings, type var(--variable-name) in any field that accepts CSS values (font size, color, padding, margin, etc).
